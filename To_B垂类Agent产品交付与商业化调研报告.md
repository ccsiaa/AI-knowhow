# To B 垂类 AI Agent 产品交付形态与商业模式调研报告

> 调研时间：2025年5月 | 调研目的：为履约智控助手的商业化路径提供行业参考

---

## 一、行业背景：2025 是垂类 Agent 商业化元年

### 1.1 宏观趋势

2025年被业界普遍视为 AI Agent 应用元年。Y Combinator 合伙人 Jared 明确指出，垂直领域 AI Agent 有望成为比 SaaS 大 10 倍的新兴市场，可能催生出市值超过 3000 亿美元的科技巨头。甲子光年智库在《2025 中国 AI Agent 行业研究报告》中断言："谁能把 Agent 做成可度量、可复购、可扩张的闭环服务，谁就能在下一个十年拿到千亿美元的船票。"

三个关键拐点同时到来：

（1）**模型成本骤降**：OpenAI o1 系列把复杂推理的成本拉低至 $0.003/1k tokens，比 2023 年 GPT-4 发布时下降 97%。这使得 Agent 在生产环境中的大规模部署首次具备经济可行性。

（2）**Agent 框架成熟**：LangGraph、CrewAI、Dify 等框架让多 Agent 协同、长时记忆在多跳任务中的错误率首次低于 5%，MCP（Model Context Protocol）协议规范了 Agent 与外部系统的交互标准。

（3）**产业认知转变**：企业客户的预期从"你给个工具"转向"你替我搞定"，合同标的从软件 License 转向 KPI 对赌 + 收益分成，行业正在经历从"交付工具"到"交付结果"的深刻范式转换。

### 1.2 市场规模

据沙利文与头豹研究院联合发布的《2025年中国AI Agent最佳实践应用榜单》，中国 AI Agent 行业已从技术探索阶段快速迈向规模化商业应用阶段。IDC 预测 2025 年全球 AI Agent 市场规模将突破 2000 亿美元，其中垂直行业解决方案占比 44.5%。民生证券认为，AI Agent 将为企业服务 SaaS 打开万亿美元蓝海市场。

## 二、产品交付形态分类

通过对市场上主流 To B 垂类 Agent 产品的调研，当前主要存在以下五种交付形态：

### 2.1 形态一：嵌入式 AI 增强（Embedded AI Enhancement）

**核心特征**：AI Agent 作为现有核心系统（如 ERP、CRM、ITSM）的增强模块，不以独立产品形态出售，而是深度集成到宿主系统中。

**代表案例**：

| 厂商 | 产品 | 嵌入宿主 | 模式说明 |
|------|------|----------|----------|
| Salesforce | Agentforce | Salesforce CRM | Agent 嵌入 CRM 平台，按对话/按动作计费 |
| ServiceNow | Now Assist AI Agent | ServiceNow ITSM | 嵌入工单系统，AI 自动处理 IT 服务请求 |
| Microsoft | Copilot Studio Agent | Microsoft 365 | 嵌入 Office/Teams 生态，按 Credit 消费计费 |
| 金蝶 | "小K"智能体 | 金蝶云苍穹 | 近 20 个智能体嵌入 ERP 全线产品 |
| 北森 | HR AI Agent | 北森 HR SaaS | 招聘/考勤 Agent 嵌入 HR 系统 |
| 衡石科技 | 嵌入式 AI Agent | 衡石 BI 平台 | 分析引擎深度集成至 SaaS 底层 |
| Samsara | Samsara Intelligence | Connected Ops Cloud | AI 嵌入物联网运营平台 |

**商业逻辑**：通过 AI Agent 提升原有产品的价值密度，驱动原产品提价（如 Salesforce 宣布 6% 涨价并附赠 Agentforce 额度），或作为高级版的差异化功能促进升级购买。AI Agent 本身并不单独销售，而是作为 SaaS 订阅的附加价值。

**适用场景**：Agent 的核心价值依赖于宿主系统的数据和流程，脱离宿主系统后无法独立运行或价值大幅缩减。

### 2.2 形态二：独立 SaaS 产品（Standalone AI Agent SaaS）

**核心特征**：AI Agent 以完整的独立 SaaS 产品形态交付，拥有自己的账号体系、UI 界面、计费系统，客户直接订阅使用。

**代表案例**：

| 厂商 | 产品 | 领域 | 模式说明 |
|------|------|------|----------|
| Harvey | Harvey AI | 法律 | 独立法律 AI Agent 平台，按席位定价 |
| 11x | Alice/Jordan | SDR 销售 | AI 数字员工独立产品，替代初级 SDR |
| Vivun | AI PreSales Agent | 售前工程 | 独立产品按 Agent 席位收费 |
| OpsVeda | Operations AI Agent | 供应链运营 | 独立运营智能平台，订阅制 |
| 零一万物 | 知识产权智能体 | IP 法律 | 独立垂直场景 Agent SaaS |
| 洞隐科技 | AI 数字员工 | 物流 TMS | 嵌入合作伙伴 TMS 但独立计费 |

**商业逻辑**：面向特定垂直场景，将 Agent 定位为"数字员工"直接替代人工岗位。客户的预算来源从 IT/软件预算转向人力预算（人力预算池通常是 IT 预算的 10 倍以上），打开了全新的定价空间。

**适用场景**：目标场景足够独立和垂直，Agent 不需要深度绑定某个特定平台的数据生态就能独立产出价值。

### 2.3 形态三：平台 + 应用市场（Platform + Agent Store）

**核心特征**：提供 Agent 构建平台和分发市场，开发者可在平台上构建垂类 Agent 并通过市场销售，平台抽佣。

**代表案例**：

| 平台 | 模式 | 抽佣比例 |
|------|------|----------|
| 字节跳动 Coze Agent Store | 开发者定价 1-99 美元/月，平台抽佣 | 30%（前 6 月让利至 20%） |
| Dify 模版市场 | 工作流模板+私有知识库，一键克隆 | 平台托管费 |
| 金蝶苍穹 Agent 平台 2.0 | 企业 Agent 生态联盟，ISV 开发专属 Agent | 生态分成 |
| 百度千帆 Agent 开发平台 | 企业级 Agent 开发+部署 | 平台消费+分成 |

**商业逻辑**：平台方不直接构建垂类 Agent，而是提供基础设施和分发渠道，通过生态聚集效应构建护城河。典型案例是 Coze Agent Store 上的"OneClickCV"简历优化 Agent，定价 $9.9/月，3 周冲到总榜第二，月收入约 25 万美元。

### 2.4 形态四：解决方案集成交付（Solution Integration）

**核心特征**：AI Agent 作为整体解决方案的一部分进行项目制交付，通常涉及定制开发、系统集成、现场部署。

**代表案例**：

| 厂商 | 产品/方案 | 领域 | 交付模式 |
|------|-----------|------|----------|
| C3.ai | C3 AI Platform + 预制应用 | 工业/能源 | 私有化部署+定制，起步约 $500K |
| Uptake（已被 Bosch 收购） | 预测性维护 Agent | 工业制造 | 深度集成客户 MES/IoT 系统 |
| 中国电信 | 政企 AI Agent | 政企 | 项目制整体交付 |
| 数商云 | 垂直行业 AI Agent | 医疗/物流 | 定制开发+私有部署 |
| Microsoft Factory Agents | Factory Safe/Operations Agent | 工业 | 与 Azure IoT 深度集成交付 |

**商业逻辑**：面向大客户，高度定制化交付，前期项目制收费（实施费+License），后续转为年度订阅维护。技术壁垒高、客单价高，但规模化复制较难。

### 2.5 形态五：结果对赌型交付（Outcome-Based Delivery）

**核心特征**：供应商对业务结果负责，根据实际达成的业务指标（ROI、成本节省额、效率提升率）进行收费。

**代表案例**：

一家私域运营 Agent 服务商为连锁烘焙品牌提供会员复购提升 Agent，合同条款为"90 天平均 ROI < 2 全额退款"。30 天内实际 ROI 达到 3.4，单店月成本仅 2000 元但带来 6800 元额外毛利。定价公式为：基础订阅费 + 超额 GMV 分成 3%-5%。

另一种变体是"数字员工外包"模式——按节省的人力成本收费。典型公式为：原客服年薪 × 替代率 × 1.5（溢价系数）。如 8 名英语客服年薪 12 万/人，Agent 替代 75%，年付费 = 8×12×0.75×1.5 = 108 万。

**商业逻辑**：最高级的商业形态，需要技术能力和业务理解都达到极高水平。供应商的信心来自于对 Agent 效果的确定性把握，客户的信任来自于"无效则无需付费"的安全感。

## 三、商业模式与定价策略深度分析

### 3.1 四大定价模式

基于对 60 余家 AI Agent 公司的研究（来源：Outreach 创始人 Manny Medina/Kyle Poyar 分析），当前市场形成了四种核心定价模式：

**模式一：按 Agent 席位定价（Per-Agent Seat）**

将 AI Agent 视为"数字员工"，固定月费部署。代表企业包括 11x、Harvey、Vivun。价值主张与员工支出直接挂钩，支出来自人力预算而非 IT 预算。优势在于可接入比技术预算大至少 10 倍的人力预算池，但面临同质化竞争和低价冲击风险。

**模式二：按动作/使用量定价（Per-Action/Consumption）**

按 Agent 实际执行的动作或消耗的资源计费。代表包括 Salesforce Agentforce（$0.10/action，即 20 Flex Credits/action）和 Microsoft Copilot Studio（按 Credit 消费）。优势是灵活、门槛低，客户按需付费；风险是收入不可预测，客户可能在试用后不持续使用。

Salesforce 的演进路径值得关注：最初按 $2/对话 定价，因对中小企业不友好而转向 Flex Credits 模式——100,000 Credits = $500，每个动作消耗 20 Credits。同时为培养使用习惯，Salesforce 向高级版客户赠送免费 Agentforce 额度。

**模式三：按工作流/流程定价（Per-Workflow）**

按 Agent 完成的完整业务流程收费，介于按动作和按结果之间。适合工作流明确、步骤可量化的场景。

**模式四：按结果/效果定价（Per-Outcome）**

客户仅在获得切实结果时付费，按照结果的一定比例收费。如 GMV 分成、节省人力成本分成。这是从客户视角最优的模式，但准确归因难度大，当前难以大规模执行。业内共识是：随着技术成熟度提升，这种模式未来最有可能成为主流。

### 3.2 定价模式与交付形态的关系

| 交付形态 | 主流定价模式 | 典型客单价 |
|----------|-------------|-----------|
| 嵌入式增强 | 原产品提价/附加消费计费 | 原订阅费 +20%-60% |
| 独立 SaaS | 按席位/按 Agent 月费 | $500-$50,000/月 |
| 平台市场 | 平台抽佣 + 开发者定价 | 平台收 20-30% |
| 解决方案集成 | 项目制 + 年度订阅 | $500K-$5M 起 |
| 结果对赌 | 基础费 + 效果分成 | 按节省成本或 GMV 分成 |

### 3.3 金蝶案例深度分析：从"卖工具"到"交付结果"的转型范式

金蝶在 2025 年的转型具有标杆意义。其策略路径为：

第一步：**AI 增强现有 SaaS**——全线产品融入 AI 能力，作为 SaaS 订阅的增值功能，维持现有商业模式的同时提升客户粘性。

第二步：**独立智能体产品**——发布五大智能体（金钥财报、ChatBI、招聘、差旅、企业知识），开箱即用，以"交付结果"为核心价值主张。

第三步：**Agent 平台生态**——苍穹 Agent 平台 2.0 + "企业管理 AI 生态联盟"，让 ISV 伙伴在平台上开发和分发垂类 Agent。

金蝶副总裁李帆明确指出："去年发布的更多属于一种'助手'，在原有 SaaS 界面有侧边栏、嵌入式的呈现，这可以称之为'AI 增强'。但今年发布的智能体是以交付结果为主，可以实现在一个完整场景中让智能化能力落地，其既可以连接金蝶内部的 SaaS 产品，也可以连接其他 SaaS 产品。"

## 四、工业/运营类垂直 Agent 产品深度调研

由于履约智控助手本质上是面向"飞行异常判断+异常修改闭环"的运营类 Agent，以下重点调研工业/运营/物流领域的相似产品。

### 4.1 OpsVeda：供应链运营 AI Agent 平台

**产品定位**：AI-Powered Operations Command Center，面向企业供应链运营的智能决策平台。

**交付形态**：独立 SaaS 平台（也通过 AWS Marketplace 分发），支持私有化部署。

**核心能力**：从异常检测到决策执行的完整闭环——实时库存监控、订单优先级调整、交付承诺更新、运力重新配置。与履约智控助手在"异常检测→判断→修正"的闭环逻辑上高度相似。

**定价模式**：订阅制，按应用数量、监控业务量和用户类型分层定价。通过 AWS Marketplace 的透明定价加速客户决策。

**商业模式闭环**：
- 价值锚点：减少运营异常导致的损失
- 入口：免费 Demo → Pilot → 全面部署
- 变现：按监控业务量的订阅费
- 增长飞轮：数据积累 → 模型优化 → 效果提升 → 续约扩容

### 4.2 C3.ai：企业级工业 AI 平台

**产品定位**：横向企业 AI 平台，但大量部署集中在工业和能源客户。

**交付形态**：平台 + 预制应用（PaaS + App）。提供预测性维护、生产优化、质量分析、库存优化、需求预测等预构建应用。

**定价模式演进**：C3.ai 经历了从传统 License（起步约 $500K）到消费制（Consumption-based）的重大转型。转型后销售周期显著缩短，市场机会扩大。这一转型启示：高客单价的项目制交付限制了规模化扩张，消费制降低了门槛但需要产品足够标准化。

**对履约智控助手的启示**：如果以独立产品出售，需要考虑是走高客单价深度交付路线（类似 C3.ai 早期），还是走标准化低门槛路线（类似 C3.ai 转型后）。

### 4.3 Samsara：Connected Operations Cloud + AI

**产品定位**：面向物理运营的 Connected Operations Cloud，覆盖车队管理、设备监控、安全合规。

**交付形态**：独立 SaaS + IoT 硬件。AI 能力（Samsara Intelligence）是平台的嵌入式增强。

**核心能力**：
- Samsara Assistant：生成式 AI 工具，即时回答复杂运营问题
- Intelligent Experiences：嵌入式 AI 驱动的智能体验
- AI Agent（2026 规划）：基于 20 万亿数据点的自主运营 Agent

**商业模式**：灵活定制定价，按组织运营规模和监控需求计费。AI 功能作为平台增值层，推动客户向更高付费层级升级。

**与履约智控助手的相似性**：同样是在既有运营系统（调度系统 vs Connected Ops Cloud）基础上提供 AI 智能运营能力，定位为运营效率的"乘数效应"。

### 4.4 工业 AI Agent 落地案例（国内）

根据 Gartner 2024 报告，超过 60% 的制造业企业已启动 AI 智能体试点项目，但仅 28% 成功实现规模化应用。国内典型案例包括：

**蒙牛**：AI Agent 在质量检测、设备预测性维护场景落地，集成到自有 MES 系统中（嵌入式增强模式）。

**振华重工**：港口设备智能运维 Agent，集成传感器数据进行故障预判和维护调度（解决方案集成模式）。

**广津物流 + 洞隐科技**：在 TMS 中集成 AI 数字员工，覆盖订单预处理、运力配载、价格审核、回单管理（嵌入式+数字员工模式）。

### 4.5 无人机物流领域 AI 运营

**Zipline**：最接近美团无人机业务的海外对标。其 AI 调度系统通过开放 API 与 Walmart 等零售巨头 ERP 系统无缝对接，实现从下单到起飞的分钟级响应。运营模式为 DaaS（Delivery as a Service），按配送次数向客户收费。智能调度系统作为 DaaS 服务的内置核心能力，不单独出售。

**禾呈羽航空**：国内无人机配送运营管理的代表。其智能调度系统实现航线规划、风控校核、运控协同的自动化与可视化。商业模式灵活性体现在"技术能力-运营能力-安全可靠性-商业模式灵活性"四维评估框架中。

## 五、商业模式闭环的关键要素

### 5.1 结果导向的价值闭环架构

成功的 To B Agent 产品普遍遵循"四层闭环"：

**感知层**：业务现状的实时量化。事件流转换为结构化数据（Embedding），核心指标为数据完整率。

**推理层**：目标拆解为可执行的 DAG（有向无环图）。Planner + 工具调用，核心指标为规划成功率。

**执行层**：Agent 调用真实系统完成操作。通过 MCP/Function Call 对接业务系统，核心指标为任务完成率。

**反馈层**：结果反向优化 Agent 能力。人类反馈+自动化评估，核心指标为 ROI 和复购率。

### 5.2 商业闭环的核心公式

无论采用哪种交付形态，商业闭环的本质是：

**客户感知价值 > 客户支付成本 > 供应商交付成本**

对于垂类运营 Agent：
- 客户感知价值 = 减少的异常损失 + 节省的人力成本 + 提升的响应速度价值
- 客户支付成本 = 订阅费/项目费/效果分成
- 供应商交付成本 = 模型调用费 + 云资源 + 人力维护

### 5.3 从 Gartner 与 IDC 视角看行业趋势

IDC 在中国 AI Agent 市场剖析中指出，企业级 Agent 正呈现四大趋势：平台化发展、混合商业模式、全球化扩张、垂直场景深耕。中国电信联合信通院发布的政企 AI Agent 白皮书也明确了"AI+、+AI、Agentic AI"三种技术融合路径。

德勤在其 Agentic AI 研究中指出："Agentic AI 不仅是一次技术迭代，更是推动组织全面进化的战略引擎。企业若能率先建立'Agent-ready'基础设施——即具备嵌入 AI、协同 AI 与治理 AI 的能力闭环——将在下一轮智能经济中抢占主动权。"

## 六、对履约智控助手商业化的分析与建议

### 6.1 履约智控助手的产品特征对标

基于对产品的理解（Web + Skill 形态，协助完成飞行异常快速判断和异常修改全闭环），其核心特征包括：

- **场景**：无人机飞行异常处理（检测→诊断→修正）
- **用户**：内部运营团队（当前）→ 外部 To B 客户（目标）
- **依赖关系**：深度依赖 DaaS 调度系统的数据和操控接口
- **技术载体**：Web 端 + Catclaw Skill 套件
- **价值闭环**：异常检测 → 根因判断 → 修正执行 → 效果验证

### 6.2 可能的商业化路径对比

| 路径 | 交付形态 | 优势 | 挑战 | 类比案例 |
|------|----------|------|------|----------|
| A：DaaS 系统增强模块 | 嵌入式增强 | 与调度系统天然耦合；降低客户决策门槛；随 DaaS 合同捆绑销售 | 无法独立定价，价值感知被稀释 | Samsara Intelligence、金蝶小K |
| B：独立产品出售 | 独立 SaaS | 独立定价空间大；可接入人力预算池；品牌独立认知 | 脱离 DaaS 后可用性存疑；客户需额外采购决策 | OpsVeda、Harvey |
| C：分层组合 | 基础嵌入+高级独立 | 兼顾两者优势；分层满足不同客户 | 产品边界需清晰划分；交付复杂度高 | Salesforce（CRM+Agentforce） |
| D：效果对赌 | 结果导向 | 客户零风险；倒逼产品能力提升 | 前期现金流压力；效果归因难 | 私域 ROI 对赌案例 |

### 6.3 核心建议

**推荐路径：C（分层组合），以 A 为基础逐步演进到 B+D 的混合模式。**

具体策略：

**第一阶段（当前→6个月）：DaaS Enhancement**

将履约智控助手定位为 DaaS 系统的"智能运维增强模块"，作为 DaaS 合同的增值包出售。定价策略为 DaaS 订阅费 +20%-40% 的增值包费用。这一阶段的核心目标是：积累外部客户使用数据、验证产品在不同客户环境下的泛化能力、建立商业化基础设施（计费、SLA、服务支持体系）。

参考金蝶 2024 年"AI 增强 SaaS"阶段的做法——在原有 SaaS 界面以侧边栏、嵌入式形态呈现 AI 能力，维持客户的使用习惯。

**第二阶段（6-18个月）：Standalone Upgrade**

当产品能力和客户基础足够成熟后，将高级功能（如全自动异常修复、预测性维护、多场景智能运营）抽离为独立产品层级。可采用"基础监控嵌入 DaaS 免费 + 高级智能运营独立订阅"的双层结构。

定价参考：
- 按 Agent 执行的"异常处理工单"数量计费（类似 Salesforce 按 Action 计费）
- 或按"节省的人工处理时间"折算定价（数字员工逻辑）

**第三阶段（18个月+）：Outcome-Based**

当效果数据充分积累后，引入效果对赌机制。如"飞行异常响应时间降低 X% 否则退费"，或按"减少的异常导致的经济损失"分成。这是最高级也是最具竞争壁垒的模式。

### 6.4 关键决策点（需确认）

1. **DaaS 系统的开放程度**：履约智控助手对 DaaS 系统 API/数据的依赖程度有多深？如果深度耦合，独立产品出售的可行性会受限。

2. **目标客户画像**：To B 客户是只购买 DaaS 的客户（如外部物流企业），还是可能有"已有自己调度系统但需要 AI 运维能力"的潜在客户？后者决定了独立产品的市场空间。

3. **效果可量化性**：飞行异常处理的效果是否可以清晰归因到智控助手？这决定了"按效果付费"模式是否可行。

4. **部署方式偏好**：客户对数据安全的敏感度如何？是否必须私有化部署？这会显著影响交付成本和商业模式选择。

## 七、结论与建议

### 7.1 核心结论

1. **To B 垂类 Agent 市场正处于爆发前夜**：2025 年被行业公认为 Agent 商业化元年，技术成本骤降、框架成熟、客户认知提升三重驱动下，垂类 Agent 产品具备规模化商业化的条件。

2. **"嵌入式增强 + 独立增值"的混合模式是最优解**：从 Salesforce、金蝶、ServiceNow 等巨头的实践来看，将 AI Agent 作为核心平台的增强能力是最快的落地路径，同时逐步孵化独立的 Agent 产品以打开增量市场。

3. **定价模式正从"卖工具"走向"卖结果"**：按效果付费、KPI 对赌是最具吸引力的模式，但需要数据积累和效果归因能力作为基础。初期宜采用订阅+用量的混合模式，逐步过渡。

4. **履约智控助手具备商业化潜力**：作为运维侧的垂类 Agent，它具有明确的价值闭环（异常检测→判断→修复）、可量化的效果指标、以及与 DaaS 系统的深度协同能力，这些都是 To B 商业化的坚实基础。

### 7.2 推荐路径

短期（0-6 个月）：作为 DaaS 系统的 AI 增强模块，纳入 DaaS 解决方案整体交付，以"AI 运维能力"作为差异化卖点，提升 DaaS 整体客单价和竞争力。

中期（6-18 个月）：在模块化成熟后，将智控助手拆分为可独立订阅的增值产品，面向已有 DaaS 客户做 Upsell，同时探索面向非 DaaS 客户的标准化产品。

长期（18 个月+）：基于积累的运维数据和效果数据，引入效果对赌/结果分成模式，从"卖工具"升级为"卖服务"，建立数据飞轮壁垒。
